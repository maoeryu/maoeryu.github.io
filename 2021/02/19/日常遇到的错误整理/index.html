日常遇到的错误整理

Posted on 2021-02-19 Edited on 2022-07-26 In 运维

日常工作中遇到的问题及解决方法.

linux

yum安装出错

解决方法

viminfo: Missing

删除/root/.viminfo

process information unavailable

有时候在centos上查看java进程时,会遇到process information unavailable的情况,
1710 -- process information unavailable

不同账号之间kill进程时,可能会造成这种现象,比如:deploy用户启动了一个java进程,但是用root用户去kill,虽然杀掉了,但是查看进程时偶尔就会出现.

解决办法

cd /tmp进到该目录,然后ls查看下,通常会遇到很多hsperfdata_开头的目录,进入可疑的目录看看,比如hsperfdata_hadoop,把1710这个给rm -rf 1710干掉,然后再看进程就消失了.

删除乱码文件

当文件名为乱码的时候,无法通过键盘输入文件名,所以在终端下就不能直接利用rm/mv等命令管理文件了.
但是每个文件都有一个i节点号,可以通过i节点号来管理文件.
首先,要取得文件的i节点号,这个可以通过ls命令的-i选项获得,

npm

initialization error

node17版本下,vue运行报错.
由于node.js v17版本中OpenSSL方面做了更新.

临时解决:设置环境变量

永久解决:降低node.js的版本不要使用v17的版本

windows_build_tools安装缓慢

.NET Framework版本过低,需要升级,下载地址:
https://dotnet.microsoft.com/en-us/download/visual-studio-sdks

下载.NET Framework 4.6.2开发包,

windows环境下,使用管理员权限安装,

su: warning: cannot change directory to /home/xxx: Permission denied

将home下xxx用户目录chown.

chown -R xxx:xxx /home/xxx

elasticsearch

kibana monitoring无数据

PUT _cluster/settings

operations are blocked on license expiration

证书过期,需要注册申请一个license.
申请地址: https://register.elastic.co/marvel_register
注册后将以email的方式发送,通过链接下载license json文件.
根据文档提示请求api并指定新的证书,

kibana索引生命周期不生效

PUT _ilm/policy/flume-monitor

PUT ztcblog*/_settings

idea

spring init timeout

使用阿里url,https://start.aliyun.com,或使用代理.

windows10

查看端口占用

1.按win+R,点击运行页面,写入cmd回车,点击命令行页面.
2.使用命令查看端口,这里查看8081端口.
netstat -aon|findstr "8081"

3.使用tasklist命令查看进程.
tasklist|findstr "20200"

4.用taskkill语句结束进程.
taskkill /f /t /im java.exe

5.用次查看,发现8081端口无被占用了.
netstat -aon|findstr "8081"
