shell-3剑客

Posted on 2021-03-03 Edited on 2022-09-06 In 运维

linux 3剑客,grep/sed/awk.

grep

grep

支持基本正则,只识别^ $ . [ ] *,其它识别为普通字符(不需要转义).

egrep

支持扩展正则,增加了( ) { } ? + |,其中(){}前需要加\转义.

fgrep

不支持正则,将特殊字符/^/$等这些正则字符识别为普通字符.

参数

参数 描述 example
-E 同egrep
-F 同fgrep
-P 将模式解释为Perl兼容的正则表达式,截取任意字符串 grep -Po '(?<=addr:).*(?= Bcast)',显示ip
-n 显示匹配的行号 grep -n text path
-c 只显示匹配的行数 grep -c text path
-i 匹配时忽略大小写,与-y类似
-v 反查,可排除多个 grep -v 'aa|bb' path
-w 全词匹配
-x 整行匹配
-r 递归,如果参数为目录时
-o 只输出匹配到的部分
-q 静默模式,无输出,需用$?判断是否执行成功,成功返回0,失败为非0
-e 匹配多个条件 grep -e text1 -e text2 -e text3 ... path
-f 将多个条件写入文件,使用-f指定该文件,与-e功能相同 grep -f xxx path
--include 包含path grep text path -r --include *.{php,html}
--exclude 排除path grep text path -r --exclude "README"
--exclude-from 排除文件列表中的文件 grep text path -r --exclude-from filelist
--exclude-dir 排除目录
-l 匹配多文件时,只显示匹配到的文件名 grep -l text path(*.log)
-L 与-l相反,只显示未匹配到的文件名
-h 查询多文件时,不显示文件名
-A/B/C num 显示匹配到的及前/后num行 grep -A/B/C num text path
-Z 指定以0值字节作为终结符文件名(\0) grep text path* -lZ |xargs -0 rm,xargs -0,读取输入并用0值字节终结符分隔文件名,然后删除匹配文件,-Z通常和-l结合使用
--color 高亮匹配到的文本 grep --color text path

grep正则

参数 描述 example
^/$ 行首/尾 grep "^text" path
| 或,多条件查找 egrep "^text|text$" path,可以有多个
\<text\> 全词匹配,与-w相同 grep "\<text\>" path
^$ 空行
^[^$] 非空行

example

sed

  • 面向行处理.
  • sed command file,其中command用单引号''括起来.
  • 如果不带file参数,则将标准输入作为处理对象.
  • 默认对原文件不做更改,只是标准输出
参数 描述 example
-n 只输出找到的行,默认多输出一遍,与p结合使用,p为打印 sed -n '/4/p' path
-e 支持多个command,多个命令之间按顺序执行 sed -n -e '1,2p' -e '4p' path
-f 将很长的command写入到文件,用-f指定该文件 sed -n -f command.path path
-i 直接修改文件,s///w输出到其它文件
-E,-r 使用扩展正则表达式

范围设定

1.指定行数

  • '3,5',表示3 4 5行
  • '3,$',表示第3行至文件最后一行

2.模式匹配
/^[^dD]/,匹配行首不是d或D开头的行

动作处理

参数 描述 example
d 删除整行
p 打印该行
r 读取指定文件的内容
w 写入指定文件
a 在下面插入新行新内容

example

参数 描述 example
y/source/dest/ 将source中的字符对位替换为dest中的字符,逐字替换,如果source部分出现重复,则只有第一次对位替换才能效果
s/regexp/replacement/ 通过正则匹配到的内容替换为replacement部分,支持&和预存储特性
'/2/d' 删除文件中带字符2的行 sed '/2/d' path
'/^$/d' 删除所有空白行
'/ *#/d; /^$/d' 删除所有注释及空白行
's/:.*$//' 以:为间隔符,提取每行的第一个域,将每行第一个:到结尾的部分清空 head -5 path | sed 's/:.*|$//'
'2,5p' 打印第2至5行 sed -n '2,5p' path
'/^[mA]/s/ao/AO/g' 将m或A开头的行中,所有的ao替换成AO,语法格式:/AA/s/BB/CC/g sed '/^[mA]/s/ao/AO/g' path
's/..$//' 删除每行最后两个字符,.代表1个字符
's/..//' 删除每行前两个字符
's/mao.*/&1988/' 将匹配到的字符串与"1988"连接,&代表之前被匹配到的部分
's/(mao)(.*)(yu)/\11988\2\31988/' 将()中的预存起来,使用\N调用,字符串连接
'/3/,/2/p' 将匹配到3(行首)和之后的第一个2(行尾)之间的打印出来 sed -n '/3/,/2/p' path
'/3/r ./aa' 将匹配到3的后面插入aa文本中的内容 sed '/3/r ./aa' path
'/3/a\hello world' 与'/3/r ./aa'命令相同 sed '/3/a\hello world' path
'/3/i\hello world' 在3前面插入
'y/ma/am/' 将所有的m替换成a,a替换成m
'/200/{n;y/eijing/EIJING/;}' 将匹配到的eijing偶数行替换成EIJING,n代表隔行替换
'/200[4-6]/w ./bb' 将匹配到的行写入指定文件中 sed '/200[4-6]/w ./bb' path

awk

符号 描述 符号 描述
$0 代表一整行的数据 FS 目前的分隔字符(字段分隔)
$1 第一个字段 OFS 输出时用指定的符号代替(字段分隔)
$2 第二个字段 RS 目前的分隔符(行分隔)
$n 第n个字段 ORS 输出时用指定的符号代替(行分隔)
NF 每行拥有的字段总数 IGNORECASE 忽略大小写
NR 目前处理的是第几行
-F fs 指定fs为分隔符,默认空白字符
-f file 从脚本文件中读取命令
-v var=var 在执行前,设置变量var,并初始化

注意

1.\n,\t需要用""双引号括起来.
2.NR,NF需要大写,且不需要$.
3.''是正则表达式的用法.

命令格式

awk '条件{命令1} 条件{命令2}...' path

控制语句

内置函数

函数 example 描述
rand awk 'BEGIN{srand(); print rand()}' 随机数
int awk 'BEGIN{srand();print int(100 rand())}' 取整
gsub awk '{gsub("l", "L", $1); print $0}' path 全局替换
sub awk '{sub("l", "L", $1); print $0}' path 只替换第一个匹配到的字符
length awk '{for (i=1; i<=NF;i++) {print $i, length($i)}}' path 返回字符串长度
substr echo "123"|awk '{print substr ($0,1,1)}' substr(string, start [, length]),截取字符串string的子串,从start开始,取length长度,若未指定长度,则返回从起始位置到字符末尾的子字符串
split awk -v ts="大娃:二娃:三娃" 'BEGIN{split (ts, huluwa, ":"); for (i in huluwa){print huluwa[i]}}' split(string, array [, fieldseq [, seps]]),将string按照分隔符fieldseq进行分割,保存在以array为名的数组中,数组下标从1开始计数,将指定字符串按照指定的分割符切割,将切割后的每一段赋值到数组的元素中,从而动态创建数组
system awk -F: 'BEGIN{system("date")}' system(command),执行系统命令,并将结果返回至awk命令,当前执行date命令并输出
systime awk 'BEGIN{now=systime(); print now }' systime()取得当前系统时间的时间戳
strfime awk 'BEGIN{now=strftime ("%D",systime());print now}' strfime()将时间戳格式化为时间,03/04/21

关键字

1.BEGIN

2.END
在读取数据后,执行END的命令.

3.printf

符号 描述

%-10s 一个宽度为10的,少的空格补齐,多的全部显示,-表示左对齐(无的话则右对齐)
%-4.2f 格式化为小数,其中.2表示保留两位小数,总共4位(整数位两位)
