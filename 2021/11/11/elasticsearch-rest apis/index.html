elasticsearch-rest apis

Posted on 2021-11-11 Edited on 2022-07-26 In 数据库

elasticsearch rest apis.

document apis

single document apis

index api

插入一条文档,索引为twitter,类型为_doc,id为1.

put twitter/_doc/1

响应,

自动创建索引

如果索引不存在,默认会自动创建索引并应用索引模板.

允许部分索引自动创建.

put _cluster/settings

关闭自动创建索引,默认为true.

操作类型

索引操作还接受op_type,可以用来强制create操作,如果索引中存在该id时,则索引操作失败.

put twitter/_doc/2?op_type=create
或使用以下方式,
put twitter/_doc/3/_create

自动生成id

不指定id时,将自动生成一个id,此时,op_type将自动设置为create.
此时,需要使用post.

post twitter/_doc/

响应,

路由

默认对文档id进行hash作为路由,可使用路由参数进行控制.
post twitter/_doc?routing=kimchy

设置mapping时,_routing可用于指导索引操作,如果设置为required时,未提供提取路由值则索引操作失败.

等待活动碎片

为了提高对系统的写入的弹性,可以将索引操作配置为在继续操作之前等待一定数量的活动碎片副本.
如果所需数量的活动碎片副本不可用,则写入操作必须等待并重试,直到所需的碎片副本已启动或发生超时.
默认情况下,写操作只等待主shards激活后再继续.
wait_for_active_shards=1

刷新

控制此请求所做的更改何时对搜索可见.
refresh

noop更新

使用索引API更新文档时,即使文档没有更改,也始终会创建文档的新版本.
detect_noop=true

超时

默认1分钟.
put twitter/_doc/1?timeout=5m

版本控制

每个索引文档都有一个版本号.
默认情况下,使用内部版本控制,从1开始,每次更新(包括删除)递增.
可使用外部值,需要启用配置version_type=true.

put twitter/_doc/1?version=2&version_type=external

使用外部版本类型时,系统会检查传递给索引请求的版本号是否大于当前存储文档的版本号.
如果为true,则文档将被索引并使用新版本号.
如果提供的值小于或等于存储文档的版本号,则会发生版本冲突,索引操作将失败.

get api

get twitter/_doc/0
当id不存在时,found为false.

响应,

实时

默认情况下,getapi是实时的,不受索引刷新率的影响(当数据在搜索中可见时).
如果文档已更新但尚未刷新,则getapi将就地发出refreshcall以使文档可见,这也将使自上次刷新以来的其他文档更改可见.
为了禁用实时获取,可以设置realtime参数为false.

源过滤

默认情况下,get操作返回_source字段,可关闭.
get twitter/_doc/1?_source=false

返回部分字段,
get twitter/_doc/1?_source_includes=*.id,user&_source_excludes=message

如果只想指定include,可使用较短表示法,
get twitter/_doc/1?_source=*.id,user

存储字段

get操作允许指定一组存储字段,这些字段将通过传递stored_fields参数,如果未存储请求的字段,则将忽略它们.

put twitter2

put twitter2/_doc/1

get twitter2/_doc/1?stored_fields=tags,counter
响应,

使用路由检索时,响应同上.

put twitter2/_doc/2?routing=user1

get twitter2/_doc/2?routing=user1&stored_fields=tags,counter

获取source

get twitter/_doc/1/_source
get twitter/_doc/1/_source?_source_includes=*.id,user&_source_excludes=message

路由

get twitter2/_doc/2?routing=user1

delete api

delete twitter2/_doc/1

update api

put test/_doc/1

脚本更新

post test/_doc/1/_update

使用脚本更新后,count值变为5.

使用脚本更新后,tag变为list,包含两个值red/blue.

使用脚本更新后,tag中blue值将变删除.

使用脚本,增加新字段.

使用脚本,移除字段.

使用脚本,如果tag中包含green,则删除文档,否则无操作.

noop

如果值无变化,则返回noop,忽略整个请求.
可增加detect_noop=false,禁用此行为.

post test/_doc/1/_update

upsert

如果文档存在则更新,不存在则插入.

post test/_doc/1/_update

doc_as_upsert

post test/_doc/4/_update

scripted_upsert

不管文档是否存在都希望脚本运行.
post sessions/session/dh3sgudg8gsrgl/_update

search apis

search

支持多索引语法.
get twitter/_search?q=user:kimchy
get kimchy,elasticsearch/_search?q=tag:wow
get _all/_search?q=tag:wow

uri search

uri中允许的参数.

名称 说明
q 查询字符串,映射到query_string
df 未定义字段前缀时使用的默认字段
analyzer 分析器名称
analyze_wildcard 是否应该分析通配符和前缀查询,默认false
batched_reduce_size
default_operator 默认运算符and/or,默认为or
lenient 设置为true时忽略文本格式,默认false
explain
_source 设置为false时禁用source字段,可使用_source_includes/_source_excludes
stored_fields 存储字段,逗号分隔
sort
track_scores
track_total_hits
timeout
terminate_after 为每个碎片收集的最大文档数,达到该数目时,查询执行将提前终止
from 要返回的命中起始索引,默认为0
size 要返回的命中数,默认为10
search_type 默认为query_then_fetch
allow_partial_search_results 默认为true,表示超时/失败时允许部分结果

get get-together/_search?q=elasticsearch&_source_includes=group.name,group.location&size=10&pretty
q后如果未指定字段,则从所有字段中搜索,elastic默认使用_all字段索引所有字段内容.

Preference

将某些搜索路由到特定的碎片副本集.

get _search?preference=xyzabc123

参数 说明
_primary 仅在主碎片上执行
_primary_first 如果可能,该操作将在主碎片上执行,但如果不执行,则将回退到其他碎片
_replica 仅在副本碎片上执行.如果存在多个副本,则未指定它们之间的优先顺序
_replica_first 如果可能,该操作将在副本碎片上执行,但如果没有,则将回退到其他碎片.如果存在多个副本,则未指定它们之间的优先顺序
_only_local 在分配给localnode的碎片上执行
_local 如果不可能的话,操作将返回到本地碎片
_prefer_nodes:abc,xyz 将在其中一个节点上执行操作
_shards:2,3 将操作限制为指定的碎片
_only_nodes:abc*,x*yz,... 将操作限制为根据指定的节点
Custom (string) value
