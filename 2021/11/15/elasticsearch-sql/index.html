elasticsearch-sql

Posted on 2021-11-15 Edited on 2022-07-26 In 数据库

elasticsearch sql,针对elasticsearch index执行SQL查询并以表格格式返回结果.

入门

创建索引,插入数据.

使用sql rest api执行sql.

post _sql?format=txt

返回结果.

使用sql cli

执行sql语句.

映射概念

sql elasticsearch 说明
column field sql存储相同类型的值,elastic可存储多种类型的值
row document
table index
schema implicit
database instance
cluster cluster

设置权限

设置用户名/密码,读/索引/管理的权限.

sql rest api

接受json文档中的sql,执行并返回结果.

post _sql?format=txt

响应数据格式

通过format属性指定.

格式 http头 说明
csv text/csv 逗号分隔值
json application/json json
tsv text/tab-separated-values 制表符分隔值
txt text/plain 类cli表示
yaml application/yaml yaml
csv

format=csv

json

format=json,响应如下,

游标

使用json格式并翻页查询时,响应会返回一个游标.
继续使用上一步得到的游标值进行查询,

post _sql?format=json

当接收最后一页时,游标自动清除,如果要清除之前的游标,使用close命令.
清除游标后,再使用游标时,将查询不到结果.

post _sql/close

过滤数据

使用query dsl过滤数据.

post _sql?format=txt

支持参数

query
要执行的sql查询.

fetch_size
默认值1000,一个响应中要返回的最大行数/条目数.

filter
默认值none,可选的查询dsl.

request_timeout
默认90s,请求失败前的超时.

page_timeout
默认45s,分页请求失败前的超时.

time_zone

field_multi_value_leniency
默认值true,当遇到一个字段的多个值时抛出异常(默认),或从列表中返回第一个值(默认升序).

注意,大多数参数(超时参数除外)仅在初始查询时有效,任何后续分页请求都只需要游标.

向查询传递参数

post _sql?format=txt

使用参数,使用?占位符,避免sql注入.

翻译sql

post _sql/translate

sql jdbc

maven

setup

主类为org.elasticsearch.xpack.sql.jdbc.EsDriver.
url为jdbc:es://[[http|https]://]*[host[:port]]*/[prefix]*<[?[option=value]&]*.

例,jdbc:es://http://server:3456/?timezone=UTC&page.size=250

参数

timezone
默认jvm timezone.

connect.timeout
默认30s,连接到server超时.

network.timeout
默认60s,network超时.

page.timeout
默认45s,等待页面超时.

page.size
默认1000,server端返回的一页的最大条目.

query.timeout
默认90s,查询超时.

user
password
field.multi.value.leniency
默认true.
