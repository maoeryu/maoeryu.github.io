hive question

Posted on 2022-06-14 Edited on 2022-12-06 In 数据库

hive日常遇到的问题.

不显示科学计数法

先将custno转化为bigint类型,然后在转化为string类型.

decimal(m,n)表示数字总长度为m位,小数位为n位,那么整数位就只有m-n位.

msck报错

外部表重建后,修复分区(msck repair table xxx)报错,如下,

因某些分区字段不规范导致,解决方法如下,
1.删除错误的文件或目录.
2.set hive.msck.path.validation=skip,跳过无效的目录.
3.set hive.msck.path.validation=ignore,跳过校验(不推荐).

修改column类型报错

hive默认不支持从高类型修改为低类型,例如bigint改为int.

解决方法:

使用关键字作为列名

将关键字用反引号括起来即可.

spark操作hive桶表

spark不支持hive桶表,需设置以下两个参数,
set hive.enforce.bucketing = false;
set hive.enforce.sorting = false;

spark创建hive外部表

spark不支持创建hive外部表,需使用hive创建.
Operation not allowed: CREATE EXTERNAL TABLE must be accompanied by LOCATION(line 1, pos 0)

获取Hive sql

获取Hive正在执行或者已结束的的MapReduce作业的SQL语句.

1)获取到作业的配置信息

通过获取接口获取运行中Hive作业可以查看到Hive的SQL语句,该信息对应到HDFS的/user/$USER/.staging/$JOBID/job.xml文件中.

通过JobHistory的API接口获取Hive历史作业执行的完整SQL语句,该信息对应到HDFS的/user/history/done/2018/02/26/000000/job_1519613953021_0029_conf.xml文件中.

2)通过hive.query.string属性过滤查看a.xml文件
查询出的sql需要用URLDecoder.decode(sql);解码.

sql错误

ERROR KeyProviderCache: Could not find uri with key [dfs.encryption.key.provider.uri] to create a keyProvider
hdfs客户端的一个bug,但并不影响作业正常运行,且在2.8版本之后已经修复.

列数对不上

FAILED: SemanticException [Error 10044]: Line 1:12 Cannot insert into target table because column number/types are different ''2021-09-18'': Table insclause-0 has 20 columns, but query has 21 columns.

检查要插入的数据列数是否与表结构相同.

parquet schema错误

Failed with exception java.io.IOException:org.apache.hadoop.hive.ql.metadata.HiveException: java.lang.UnsupportedOperationException: Cannot inspect org.apache.hadoop.io.LongWritable

底层文件存储采用 parquet,hive 建表语句中的字段类型与 parquet 文件中的类型不一致导致.
查看 parquet 文件 schema,重建表或修改列类型.

hudi集成错误

cannot find class org.apache.hudi.hadoop.HoodieParquetInputFormat.
添加hudi-hadoop-mr-bundle-0.12.0.jar至lib/后,
配置hive第三方jar路径,hive.aux.jars.path
