hive中实现group_concat

mysql中的group_concat分组连接功能相当强大,可以先分组再连接成字符串,还可以进行排序连接.
hive中并没有这个函数,这里要用到:concat_ws/collect_list/collect_set 函数.

建立测试表(无分区表):

插入几条数据:

id content comment
1 Wendy 测试3
2 Bob 测试22
2 Tom 测试11
1 Bob 测试2
1 Tom 测试1

concat_ws + collect_set + group by

id con_con con_com
1 Tom,Bob,Wendy 测试3,测试1,测试2
2 Tom,Bob 测试22,测试11

结果:无序且不对应(con_con与con_com的位置)
collect_set会将重复的数据删除,因为集合的性质.
每次运行连接的结果顺序都可能不一样.

concat_ws + collect_list + group by

id con_con con_com
1 Wendy,Bob,Tom 测试3,测试2,测试1
2 Bob,Tom 测试22,测试11

结果:对应(con_con与con_com的位置)但无序.

concat_ws + collect_list + group by + row_number()

id content comment rn
1 Bob 测试2 1
1 Tom 测试1 2
1 Wendy 测试3 3
2 Bob 测试22 1
2 Tom 测试11 2
id con_con con_com con_rn
1 Bob,Tom,Wendy 测试2,测试1,测试3 1,2,3
2 Bob,Tom 测试22,测试11 1,2

结果:对应(con_con与con_com的位置)且有序.
