检查提交

更新插入或插入一批传入记录的任务在 Hudi 中称为提交.
提交提供了基本的原子性保证,因此只有提交的数据可用于查询.
每个提交都有一个单调递增的字符串/数字,称为提交号.
通常,这是我们开始提交的时间.

您可以使用那里的时间戳来估计提交已经进行了多长时间.

归档提交

为了限制 DFS 上 .commit 文件的增长量,Hudi 将较旧的 .commit 文件(在适当考虑清洁策略的情况下)归档到 commits.archived 文件中.
这是一个序列文件,其中包含来自 commitNumber => json 的映射,其中包含有关提交的原始信息.

Hudi CLI 还提供手动升级 Hudi 表的功能.
通过 CLI 升级 Hudi 表:

升级Hudi库后,在DeltaStreamer等不同部署模式下,表升级由Hudi写客户端自动处理,用户无需手动升级.
这种自动表升级是一般推荐的方式,而不是使用upgradeCLI 命令.

从表版本 ONE 升级到 TWO 表需要密钥生成器相关配置,例如"hoodie.datasource.write.recordkey.field",该配置仅在用户配置写入作业时可用.
因此不支持通过 CLI 将表从版本 ONE 升级到 TWO,而用户应依赖写入客户端中的自动升级.

您也可以在不指定目标版本的情况下运行升级命令.
在这种情况下,使用与库发布版本对应的最新表版本:
upgrade table
