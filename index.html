hive元数据表

Posted on 2022-07-28 In 数据库

概述

Hive的元数据信息在MySQL数据中有57张表.

版本

VERSION

存储hive版本的元数据表.
ID主键|Hive版本|版本说明

如果该表出现问题,根本进入不了Hive-Cli.
比如该表不存在,当启动Hive-Cli时候,就会报错"Table 'hive.version' doesn't exist".

数据库相关

DBS和DATABASE_PARAMS这两张表通过DB_ID字段关联..

DBS

存储Hive中所有数据库的基本信息.
数据库ID|数据库描述|数据库HDFS路径|数据库名|数据库所有者用户名|所有者角色

DATABASE_PARAMS

该表存储数据库的相关参数,在CREATE DATABASE时候用WITH DBPROPERTIES (property_name=property_value, ...)指定的参数.
数据库ID|参数名|参数值

表/视图

主要有TBLS/TABLE_PARAMS/TBL_PRIVS,这三张表通过TBL_ID关联.

TBLS

该表中存储Hive表/视图/索引表的基本信息.
表ID|创建时间|数据库ID|上次访问时间|所有者|保留字段|序列化配置信息|表名|表类型|视图的详细HQL语句|视图的原始HQL语句

其中,DB_ID对应DBS中的DB_ID,SD_ID对应SDS表中的SD_ID.
TBL_TYPE有MANAGED_TABLE/EXTERNAL_TABLE/INDEX_TABLE/VIRTUAL_VIEW.

TABLE_PARAMS

该表存储表/视图的属性信息.
表ID|属性名|属性值

TBL_PRIVS

该表存储表/视图的授权信息.
授权ID|授权时间||授权执行用户|授权者类型|被授权用户|被授权用户类型|权限|表ID

其中TBL_ID对应TBLS表中的TBL_ID.

文件存储信息

主要涉及SDS/SD_PARAMS/SERDES/SERDE_PARAMS.
由于HDFS支持的文件格式很多,而建Hive表时候也可以指定各种文件格式,Hive在将HQL解析成MapReduce时候,需要知道去哪里,使用哪种格式去读写HDFS文件,而这些信息就保存在这几张表中.

SDS

该表保存文件存储的基本信息,如INPUT_FORMAT/OUTPUT_FORMAT/是否压缩等.
TBLS表中的SD_ID与该表关联,可以获取Hive表的存储信息.
存储信息ID|字段信息ID|文件输入格式|是否压缩|是否以子目录存储|HDFS路径|分桶数量|文件输出格式|序列化类ID

其中CD_ID对应CDS表,SERDE_ID对应SERDES表.

SD_PARAMS

该表存储Hive存储的属性信息,在创建表时候使用STORED BY 'storage.handler.class.name' [WITH SERDEPROPERTIES (...)指定.
存储配置ID|存储属性名|存储属性值

SERDES

该表存储序列化使用的类信息
序列化类配置ID|序列化类别名|序列化类

SERDE_PARAMS

该表存储序列化的一些属性/格式信息,比如:行/列分隔符.
序列化类配置ID|属性名|属性值

表字段

COLUMNS_V2

该表存储表对应的字段信息.
字段信息ID|字段注释|字段名|字段类型|字段顺序

表分区

主要涉及PARTITIONS/PARTITION_KEYS/PARTITION_KEY_VALS/PARTITION_PARAMS.

PARTITIONS

该表存储表分区的基本信息.
分区ID|分区创建时间|最后一次访问时间|分区名|分区存储ID|表ID

PARTITION_KEYS

该表存储分区的字段信息.
表ID|分区字段说明|分区字段名|分区字段类型|分区字段顺序

PARTITION_KEY_VALS

该表存储分区字段值.
分区ID|分区字段值|分区字段值顺序

PARTITION_PARAMS

该表存储分区的属性信息.
分区ID|分区属性名|分区属性值

其它

DB_PRIVS

数据库权限信息表.
通过GRANT语句对数据库授权后,将会在这里存储.

IDXS

索引表,存储Hive索引相关的元数据.

INDEX_PARAMS

索引相关的属性信息.

TAB_COL_STATS

表字段的统计信息.
使用ANALYZE语句对表字段分析后记录在这里.

TBL_COL_PRIVS

表字段的授权信息.

PART_PRIVS

分区的授权信息.

PART_COL_STATS

分区字段的统计信息.

PART_COL_PRIVS

分区字段的权限信息.

FUNCS

用户注册的函数信息.

FUNC_RU

用户注册函数的资源信息.

yarn-site.xml

添加新节点

1.配置运行环境
安装与之前节点相同的hadoop/java等环境,配置ssh免密登录.

2.修改配置
修改集群所有节点(master和所有slave)的slaves文件,增加新节点ip或hostname.
同时,集群所有节点上修改系统的/etc/hosts配置文件,增加ip与hostname的映射.

3.刷新节点
hdfs dfsadmin -refreshNodes
yarn rmadmin -refreshNodes

4.启动新节点
sbin/hadoop-daemon.sh start datanode
sbin/yarn-daemon.sh start nodemanager

5.检查webui.

退役节点

1.添加地址到exclude,但不要更新include.
2.刷新节点,
hdfs dfsadmin -refreshNodes
yarn rmadmin -refreshNodes

3.进入webui,查看节点状态是否为"Decommission In Progress",开始复制数据块到其他节点.
4.当所有退役中的节点报告为"Decommissioned",说明所有块已经复制完成,退役节点即可停机.
5.从include中删除节点地址.
6.刷新节点
hdfs dfsadmin -refreshNodes
yarn rmadmin -refreshNodes

7.从slaves文件删除节点.

解决办法的突破点就在于如何把左表的未关联记录的key尽可能打散,因此可以这么做:
若左表关联字段无效(为空/字段长度为零/字段填充了非整数),则在关联前将左表关联字段设置为一个随机数,再去关联右表.
这么做的目的是即使是左表的未关联记录,它的key也分布得十分均匀.

break/continue/return区别

break

1.结束当前整个循环,执行当前循环下边的语句.忽略循环体中任何其它语句和循环条件测试.
2.只能跳出一层循环,如果你的循环是嵌套循环,那么你需要按照你嵌套的层次,逐步使用break来跳出.[逐层判断,逐步跳出]
3.break在循环体内,强行结束循环的执行,也就是结束整个循环过程,不在判断执行循环的条件是否成立,直接转向循环语句下面的语句.
4.当break出现在循环体中的switch语句体内时,其作用只是跳出该switch语句体.
总的来说：就近原则,结束当前的整个循环.

return

1.从当前的方法中退出,返回到该调用的方法的语句处,继续执行.

maven使用

Posted on 2022-07-27 In 运维

打包依赖

scope详解

compile

默认就是compile,什么都不配置也就是意味着compile.
compile表示被依赖项目需要参与当前项目的编译,当然后续的测试,运行周期也参与其中,是一个比较强的依赖.
打包的时候通常需要包含进去.

test

scope为test表示依赖项目仅仅参与测试相关的工作,包括测试代码的编译,执行.
比较典型的如junit.

runntime

runntime表示被依赖项目无需参与项目的编译,不过后期的测试和运行周期需要其参与.
与compile相比,跳过编译而已,在终端的项目中,和compile区别不是很大.
比较常见的如JSR×××的实现,对应的API jar是compile的,具体实现是runtime的,compile只需要知道接口就足够了.
oracle jdbc驱动架包就是一个很好的例子,一般scope为runntime.
另外runntime的依赖通常和optional搭配使用,optional为true.

provided

provided意味着打包的时候可以不用包进去,别的设施(Web Container)会提供.
事实上该依赖理论上可以参与编译,测试,运行等周期.
相当于compile,但是在打包阶段做了exclude的动作.

system

从参与度来说,也和provided相同,不过被依赖项不会从maven仓库抓,而是从本地文件系统拿,一定需要配合systemPath属性使用.

scope的依赖传递

A–>B–>C.
当前项目为A,A依赖于B,B依赖于C.
知道B在A项目中的scope,那么怎么知道C在A中的scope呢?答案是:
当C是test或者provided时,C直接被丢弃,A不依赖C;
否则A依赖C,C的scope继承于B的scope.
