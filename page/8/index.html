hive_hbase集成

Posted on 2021-03-02 Edited on 2021-03-05 In database

概述

hive和hbase在大数据架构中处在不同位置.

hive

hive是一个构建在hadoop基础之上的数据仓库,主要解决分布式存储的大数据处理和计算问题.

hive提供了类sql语句(hiveql),可以查询存放在hdfs上的数据,sql语句最终被转化为map/reduce任务运行,适合用来对离线数据进行分析查询.

hbase

hbase是hadoop database的简称,是一种nosql数据库,非常适用于海量明细数据(十/百亿)的随机实时查询,如交易清单/轨迹行为等.

整合

在大数据架构中,hive和hbase是协作关系,hive方便地提供了hive ql的接口来简化mapreduce的使用, 而hbase提供了低延迟的数据库访问.

如果两者结合,可以利用mapreduce的优势针对hbase存储的大量内容进行离线的计算和分析.

hive语言手册

Posted on 2021-02-26 Edited on 2022-07-05 In database

set hiveconf:hive.cli.print.current.db; //打印当前数据库名
set hive.exec.mode.local.auto=true; //使用本地模式加快查询速度
set hive.cli.print.header; //打印字段头
set hive.map.aggr=true; //在map端优先聚合,需要更多内存
set hive.exec.mode.local.auto=true; //默认false,使用本地模式执行,避免mr
set hive.exec.mode.local.auto.input.files.max=4; //
set hive.exec.mode.local.auto.inputbytes.max=134217728; //job总输入大小
set hive.mapred.local.mem=0; //默认0,由hadoop确定子jvm的默认内存限制
set hive.exec.parallel=false; //默认false,设置为true,并行执行多个阶段
//调整map和reduce个数
set hive.exec.reducers.bytes.per.reducer=256000000; //默认256m
set hive.exec.reducers.max=1009; //集群中map和reduce个数
set mapreduce.job.reduces=-1; //
//压缩
set hive.exec.compress.intermediate=true; //默认false,开启中间压缩,减少数据传输量
set hive.exec.compress.output=true; //默认false,最终结果压缩
set hive.optimize.correlation=true; //开启关联优化
set hive.groupby.skewindata=false; //默认false,表示不存在数据倾斜
set hive.execution.engine=mr; //默认mr,可改为spark

redis中bitmap的使用

Posted on 2021-02-22 Edited on 2021-03-05 In database

通过一个bit位来表示某个元素对应的值或状态,每个bit的值只能是0或1.

8个bit可以组成一个byte,所以bitmap极大的节省储存空间.

bitmap不是特殊的数据结构,它是普通的字符串(byte数组),可以用get/set操作,也可以使用位图操作getbit/setbit等将byte数组当成位数组来处理.

位数组是自动扩展的,如果设置了某个偏移位置超出了现有的内容范围,就会自动扩充.
